## 1. 비즈니스 핵심 소개

**서비스명**: 건기식 성분·가격 비교 초자동화 플랫폼 (Super-Calc)  
**엘리베이터 피치**: *"수동 엑셀 계산과 뒷광고 필터링에 지친 건기식 소비자를 위한, 리얼-타임 1일 단가 계산 및 의학 팩트체크 플랫폼"*

### 타겟 문제 (Lemon Market)
- 연 **6조 원** 시장에 수천 개 브랜드가 난립 → 정보 과잉·비교 불가 상태
- 동일 성분 기준 가격 차이 **최대 8.2배**, 소비자 47.2%가 성분 비교 어려움 호소
- 탐색~결제 **60분 이상** 소요, 여정 중간 이탈률 **55~75%**

### 솔루션 — 4대 핵심 기능
| Feature | 한 줄 요약 |
|---------|-----------|
| **Super-Calc Engine** | iHerb·쿠팡·네이버 3채널의 1일 복용량 기준 원화 최종 단가를 **5초** 안에 자동 비교 |
| **Anti-BS Dashboard** | 광고·협찬 콘텐츠 **100% 제거**, 식약처 공전 기반 의학 뱃지(✅/⚠️/🚫)만 표시 |
| **Viral Engine** | 비교 결과를 OG 이미지 포함 카카오톡 공유 카드로 **1탭** 전송, 앱 설치 불요 |
| **Data Trust System** | 모든 데이터의 원본 출처를 **2클릭** 이내 추적, 오류 제보 시 **48h SLA** 수정 보장 |

## 2. 고객 핵심가치 (Value Proposition)

**포지셔닝 선언**: 마케팅 노이즈를 100% 제거하고 오직 **'1정당 찐 단가'** 와 **'식약처/논문 뱃지'** 만을 제공하는 시장 내 유일한 **독립(Neutral) 플랫폼**.

| 고객 Pain | 기존 대안 | 우리의 제안 | 결과 지표 |
|-----------|----------|------------|----------|
| 채널별 단가 수동 계산 60분 | 개인 스프레드시트 | Super-Calc 자동 환산 | **60분 → 5초** (720배 단축) |
| 광고성 콘텐츠 72.3% 범람 | 블로그·유튜버 탐색 | Anti-BS 마케팅 노이즈 제로 UI | 광고 노출 **0건** |
| 성분명 해석 불가 47.2% | 약국 방문·지인 추천 | 전문 용어 일상어 번역 + 뱃지 | 이해도 **95%↑** |
| 데이터 오류 불신 (기존앱 10%+) | 직접 라벨 확인 | 출처 투명 공개 + 48h 수정 SLA | 오류율 **≤5%** |

**차별적 해자**: 비표준 성분 표기를 통일하는 **데이터 정규화(Normalization)** 기술력 — 후발 복제가 어려운 시간 집약적 기술 자산.

## 3. Core Persona · JTBD · 고객여정(CJM)

### 3-1. C1 한정훈 (36, 개발자) — 수익 엔진 / Q1-A 가성비 최적화자
- **JTBD**: *"환율·배송비·복용량이 반영된 1일 단가를 5초 안에 비교하고 즉시 결제"*
- **Switch Trigger**: 수동 계산의 피로 + 실시간 반영 실패가 만나는 지점
- **대표 인용**: *"엑셀에 달러치고 나눗셈 하다보면 현타옵니다."*

| CJM 단계 | 행동 | 감정 | Pain → 기회 |
|----------|------|------|-------------|
| 인지 | "함량당 단가 비교 사이트" 커뮤니티 접촉 | 반신반의 | 도구 존재 자체를 모름 → SEO 선점 |
| 고려 | 본인 비타민D로 기능 테스트, 환율 반영 검증 | 비판적 집중 | 데이터 정확도 직접 검증 수단 필요 → 출처 투명 공개 |
| 결정 | 계산값 일치 확인 → "40분이 3초네. 쓴다." | **즉각 확신** | 최종가(배송비·쿠폰) 미반영 시 이탈 → 실질 최저가 표시 |
| 충성도 | 2~3개월 주기 재방문, 커뮤니티 자발 공유 | 루틴 정착 | DB 커버리지 미흡 시 이탈 → 제품 요청 기능 |

### 3-2. C2 박소연 (43, 인사팀 과장) — 성장 엔진 / Q4-A 건강 계기 진입자
- **JTBD**: *"광고 배제 후 객관적 팩트만 필터링하여 30분 내 확신 있는 결정, 자신 있게 공유"*
- **Switch Trigger**: 블로그 광고 피로 + 성분 용어 해석 불가 + 결론 부재
- **대표 인용**: *"광고인지 아닌지만 감별해 주는 판독기가 필요해요."*

| CJM 단계 | 행동 | 감정 | Pain → 기회 |
|----------|------|------|-------------|
| 인지 | 건강검진 비타민D 부족 → "비타민D 추천" 검색 | 불안+혼란 | 광고 vs 독립정보 구분 불가 → "광고 아님" 정체성 명확화 |
| 고려 | 성분명 일상어 번역 읽기 시도, 가격 적정성 확인 | 기대 반 불안 반 | 성분명 이해 불가 → 일상어 번역 + "적정 가격" 표시 |
| 결정 | "광고 아닌 곳에서 이렇게 나왔으면 믿어도 되겠지" | 조심스런 안도 | 마지막 의심 → 독립 비교 선언 배지 |
| 충성도 | Q4→Q1 전환(6~12개월), 가족 영양제도 검색 | **자신감·전도사** | 전문가 모드 부재 시 이탈 → 성장 UX |

### 3-3. 성장 플라이휠
```
A2(트렌드 탐색자) → SNS 비교 카드 공유 → C2 첫 유입
C2 → 반복 사용 6~12개월 → Q4→Q1 전환 → C1
C1 → 제휴 링크 클릭 → 매출 (SOM 수익 55%)
C1/C2 → 카카오 비교 카드 → E1(부모님)·N1(지인) 간접 노출
```

## 5. 원하는 CTA — 사전 예약 / 알림 설정 / 대기 리스트 등록

### CTA 전략 설계

| CTA 유형 | 문구 예시 | 배치 위치 | 타겟 페르소나 |
|----------|----------|----------|-------------|
| **사전 예약** | "출시 알림 받기 — 가장 먼저 써보세요" | 히어로 섹션 (주력) | C1 (빠른 전환, 도구 갈망) |
| **대기 리스트** | "대기 리스트 등록 — 이메일만 남기면 끝" | Result Gallery 하단 | C2 (낮은 진입 장벽 필수) |
| **알림 설정** | "알림 설정하기 — 오픈 즉시 알려드릴게요" | 최종 CTA 섹션 | A2 (FOMO 활용, 바이럴 연결) |

### CTA 설계 원칙
1. **이메일만 수집** — 회원가입 마찰 제로 (C2/E1 진입 장벽 고려)
2. **얼리버드 혜택 명시** — "사전 등록 시 론칭 후 3개월 프리미엄 무료" 등
3. **숫자 카운터 노출** — "현재 N명이 대기 중" → 사회적 증거(Social Proof) 강화
4. **반복 배치** — 스크롤 기준 상단·중단·하단 최소 3회
5. **페르소나별 미세 조정**:
   - C1 대상: "엑셀 비교를 영원히 해고하세요" → 도구 대체 소구
   - C2 대상: "30분 안에 확신 있는 선택, 곧 만나보세요" → 안심 소구
   - A2 대상: "트렌드 성분 팩트체크, 제일 먼저 공유하세요" → FOMO 소구
