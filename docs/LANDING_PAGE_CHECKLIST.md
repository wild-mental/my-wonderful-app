## v1 → v2 변경 요지

기존 v1 랜딩(검색·결과 지향)은 **이미 출시된 도구**처럼 보이게 했습니다.
v2는 **프리런치 매니페스토**로 전면 재포지셔닝하여, 사전예약/대기리스트/알림이라는 3-tier 이메일 캡처를 핵심 전환 장치로 사용합니다.

| 영역 | v1 (Result-First) | **v2 (Manifesto-First)** |
|------|-------------------|--------------------------|
| 헤드라인 | "영양제, 진짜 가격을 3초 만에" | **"광고 없는 팩트, 엑셀 없는 최저가."** |
| 톤 | 친절한 안내자 | 단호한 매니페스토 ("우리는 ~을 하지 않습니다") |
| 첫 인상 | 검색바 → /app 진입 유도 | 다크 매니페스토 + 라이브 대기 카운터 |

| 유형 | 설명 | 선택 |

### 하이브리드 채택 근거
- 건기식 시장은 **레몬 마켓**이며, 소비자가 광고와 정보를 구분하지 못하는 **'신뢰 결여'** 가 1차 Pain → A 유형
- 동시에 60분 수동 엑셀 비교를 5초 자동화로 **'결과 단축'** 시키는 것이 도구로서의 본질 → C 유형
- 두 페르소나(C1 가성비 최적화자 / C2 신뢰 탐색자)가 각기 다른 유형의 Pain을 가짐 → 단일 유형으로 풀 수 없음

## 데이터 영속화 명세

```ts
localStorage["supercalc_waitlist"] = {
  "emails": ["a@b.com", "c@d.com", ...] // unique set
}
```

- 이메일 정규식: `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
- 중복 자동 dedupe (Set)
- 페이지 재로드 시 baseCount(2,837) + 저장된 이메일 수 = 시작 카운터
- 탈퇴/리셋: `localStorage.removeItem("supercalc_waitlist")`
